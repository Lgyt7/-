# Учебная практика: Оптимизация поиска в массиве

## Цель и задачи

**Цель:** Освоить методику измерения производительности кода, выявления «узких мест» и ускорения программ за счёт замены неэффективных алгоритмов более быстрыми структурами данных и встроенными функциями PHP.

**Задачи:**
1. Запустить предоставленный «медленный» скрипт поиска товара в массиве.
2. Измерить время его выполнения.
3. Выявить причину низкой скорости (линейный перебор всего массива).
4. Переписать поиск с использованием ассоциативного массива и доступа по ключу.
5. Сравнить производительность «до» и «после» с помощью модульного бенчмарка.

---

## Исходный код (медленный вариант)

```php
function generateProducts(int $count): array {
    $products = [];
    for ($i = 0; $i < $count; $i++) {
        $products[] = [
            'id' => $i,
            'name' => 'Product_' . $i,
            'price' => rand(100, 10000)
        ];
    }
    return $products;
}

function searchProductSlow($products, $searchId) {
    foreach ($products as $product) {
        if ($product['id'] == $searchId) {
            return $product;
        }
    }
    return null;
}
```
Причина медленной работы: полный перебор всех 50 000 элементов для каждого вызова функции. Даже если искомый элемент находится в начале, в среднем и худшем случаях выполняется обход всего массива.

Оптимизированное решение

```php
function searchProductFast($products, $searchId) {
    // Преобразуем массив в ассоциативный, где ключ — id товара
    $assoc = [];
    foreach ($products as $product) {
        $assoc[$product['id']] = $product;
    }
    // Доступ к элементу за O(1)
    return $assoc[$searchId] ?? null;
}
```

Пояснение:
Вместо линейного сканирования мы однократно проходим по массиву, создавая новый, индексированный по id. После этого поиск любого элемента выполняется через доступ по ключу, что имеет константную сложность O(1).

Альтернативный подход (оставлен в комментариях кода): использование array_column() + array_search(). Он тоже быстрее чистого цикла, но всё ещё обладает сложностью O(N) и уступает индексированному доступу.

Функция бенчмаркинга
Для объективного сравнения реализована универсальная функция runBenchmark, которая:

принимает тестируемую функцию, данные, искомый ID и число повторений;

замеряет общее время выполнения заданного количества вызовов;

возвращает общее время в секундах, среднее время в миллисекундах и количество итераций.

Код функции:

```php
function runBenchmark(callable $function, array $data, int $searchId, int $iterations = 100): array {
    $start = microtime(true);
    for ($i = 0; $i < $iterations; $i++) {
        $function($data, $searchId);
    }
    $finish = microtime(true);
    
    $totalTime = $finish - $start;
    $avgTimeMs = ($totalTime / $iterations) * 1000;
    
    return [
        'total_time_sec' => round($totalTime, 6),
        'avg_time_ms'    => round($avgTimeMs, 6),
        'iterations'     => $iterations
    ];
}
```

Выводы
Линейный перебор массива (foreach) – простой, но крайне неэффективный способ поиска для больших массивов.

Преобразование массива в ассоциативный с ключом по идентификатору позволяет получать элементы за константное время, радикально повышая производительность.

Для разовых поисков можно применять array_column() и array_search(), но если поиск выполняется многократно, наилучшим решением является однократная индексация данных.

Модульное тестирование производительности с помощью функции-бенчмарка позволяет точно измерить ускорение и объективно сравнить разные реализации.

Полученные навыки оптимизации применимы к любым проектам на PHP, где требуется быстрый доступ к элементам коллекций.
